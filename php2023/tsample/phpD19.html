php 부분만 Prepared Statements를 이용하여 다시 한번 작업해보겠습니다. 

<?php
    include "../connect/connect.php";
    include "../connect/session.php";
    include "../connect/sessionCheck.php";

    // POST 데이터 처리
    $boardTitle = $_POST['boardTitle'] ?? '';
    $boardContents = $_POST['boardContents'] ?? '';
    $memberID = $_SESSION['memberID'] ?? '';

    // 데이터 입력 여부 확인
    if(empty($boardTitle) || empty($boardContents)){
        echo "<script>alert('제목 또는 게시글을 작성해주세요!'); history.back();</script>";
        exit;
    }

    $boardView = 1;
    $boardDelete = 1;
    $regTime = time(); 

    // SQL 준비된 명령문을 사용하여 데이터베이스 쿼리 실행
    $stmt = $connect->prepare("INSERT INTO board(memberID, boardTitle, boardContents, boardView, boardDelete, regTime) VALUES(?, ?, ?, ?, ?, ?)");
    $stmt->bind_param("issiii", $memberID, $boardTitle, $boardContents, $boardView, $boardDelete, $regTime);

    $result = $stmt->execute();

    if($result){
        echo "<script>alert('게시글이 성공적으로 작성되었습니다.'); window.location.href = 'board.php';</script>";
    } else {
        // 오류 메시지를 로그 파일에 기록하거나 개발자에게 알리는 등의 조치 필요
        error_log("Error: " . $stmt->error);
        echo "<script>alert('게시글 작성에 오류가 있습니다. 관리자에게 문의하세요'); window.location.href = 'board.php';</script>";
    }

    $stmt->close();
?>

Prepared statements는 PHP와 같은 프로그래밍 언어에서 데이터베이스와 상호 작용할 때 사용되는 중요한 개념 중 하나입니다. Prepared statements는 SQL 쿼리를 안전하게 실행하고 SQL 인젝션 공격을 방지하는 데 도움이 되는 방법입니다.

  • SQL 인젝션 방지: Prepared statements는 사용자로부터 입력된 데이터를 안전하게 처리하는 방법입니다. 일반적으로 SQL 쿼리에 사용자 입력을 직접 포함시키면 악의적인 사용자가 악용하여 데이터베이스를 공격할 수 있습니다. Prepared statements를 사용하면 입력 데이터가 자동으로 이스케이핑되고 쿼리의 매개 변수로 처리되므로 SQL 인젝션 공격을 피할 수 있습니다.
  • 성능 개선: Prepared statements를 사용하면 동일한 쿼리를 여러 번 실행해야 할 때 성능을 향상시킬 수 있습니다. 쿼리를 미리 컴파일하고 실행 시에 매개 변수만 변경하면 됩니다. 이로 인해 데이터베이스 서버에서 쿼리를 다시 컴파일하는 오버헤드를 줄일 수 있습니다.
  • 코드 가독성: Prepared statements는 SQL 쿼리와 입력 데이터를 분리시키므로 코드가 더 읽기 쉽고 유지 관리하기 쉬워집니다.
