소개

이제 정말 마무리 단계에 접어들었습니다. 스크롤 할 때 스므스 효과를 주면 조금더 고급지게 만들 수 있습니다. 물론 웹사이트를 만들 때 자주 사용되는 탭 메뉴, 이미지 슬라이드, 로딩 처리 등을 구현하지 않은 부분이 여전히 많습니다. 이런 부분도 다 알려드리고 싶지만, 너무 많은 내용을 한 번에 전달하기보다는 조금씩 천천히 반복적으로 학습하는 것이 가장 중요합니다.

잘하는 것보다 오래동안 꾸준히 하는게 더 중요하더라구요! 흥미를 느끼면서 오랜 기간 동안 노력한다면, 능숙하게 웹 개발을 할 수 있을 것입니다. 포기하지 않고 최선을 다하면 여러분들도 훌륭한 개발자가 될 수 있습니다. 그럼 마지막까지 열심히 마무리 해봅시다! 🚀

10. 마무리

10.1 스무스 효과주기

셋팅에서 lenis를 설치하였기 때문에 바로 작업하면 됩니다. 소스는 사이트 웹 문성에 잘 나왔기 때문에 복사해서 사용하겠습니다. main.js에서 수정하겠습니다.

Lenis를 import 하고 브라우저가 다 로드 했을 때 스므스 효과를 실행합니다. 메뉴와 포트폴리오 스크립트는 작동할 수 있도록 실행함수를 설정하면 됩니다. 이렇게 설정하고 console.log를 확인하면 수시로 움직이는 걸 확인할 수 있습니다. 

import { menu } from "./menu.js";
import { port } from "./port.js";

import Lenis from "@studio-freight/lenis";

window.addEventListener("load", function () {
    // 스므스 효과
    const lenis = new Lenis({
        duration: 1,
        easing: (t) => Math.min(1, 1.001 - Math.pow(2, -10 * t)),
    });

    function raf(time) {
        lenis.raf(time);
        requestAnimationFrame(raf);
    }

    requestAnimationFrame(raf);

    lenis.on("scroll", (e) => {
        console.log(e);
    });
    
    menu();
    port();
});

이런 부분이 보이면 제대로 작동하고 있는 겁니다.

10.2 빌드하고 github에 올리기

배포하는 방법은 여러가지가 있지만 우선 첫 시간에 배웠던 git에 모든 소스를 올려야 합니다. 그전에 우리가 작업한 파일을 조금 더 가볍게 만들기 위해서 번들링 작업을 해야 합니다. 소스에 빈칸이 없고 한줄로 만드는 효과 보신적 있나요? 이렇게 하면 소스 용량도 줄어들고 가벼워집니다. 이런 부분은 webpack을 통해서 작업하지만 vite는 이런 부분을 매우 잘 해줍니다. 그래서 먼저 빌드를 통해 소스를 간단 명료하게 만들어주겠습니다. 

npm run build

10.3 netlify에 배포하기

github에 무사히 올렸다면 www.netlify.com 사이트에 회원가입을 하겠습니다. 웬만하면 github 아이디로 로그인하면 좋습니다. 바로 연동이 되니까 github 아이디로 로그인 하겠습니다. 로그인을 했다면 해당 부분을 클릭하면 됩니다. 기존에 만들었던 사이트 들이 있어서 보이는 모습이 틀릴 수 있지만 거의 비슷할 겁니다. 첫번재 메뉴를 클릭하고 github을 클릭하면 내 저장소로 이동합니다.

우리가 만들었던 저장소를 선택하면 됩니다.

마지막으로 빌드 이름만 public으로 변경합니다.

완성이 되면 주소가 나옵니다. 주소가 너무 길기 때문에 domain setting에 option으로 들어가서 수정을 합니다. 저는 https://port2023-html.netlify.app 이렇게 수정했습니다.
