2_1. 라우팅 설정하기

react-router-dom은 React.js 기반의 웹 애플리케이션에서 라우팅을 관리하기 위한 라이브러리입니다. React 애플리케이션을 단일 페이지 애플리케이션(SPA)으로 만들 때, 여러 페이지 간의 전환과 URL 기반 라우팅을 구현하는 데 사용됩니다. 이를 통해 사용자가 웹 애플리케이션을 사용하는 동안 새로운 페이지를 로드하지 않고도 다른 컴포넌트를 렌더링하여 사용자 경험을 향상시킬 수 있습니다. 즉 리액트는 한페이지로 구성된 SPA이기 때문에 주소 페이지가 없습니다. 해당 컴퍼넌트만 불러오기 때문에 이때 페이지가 있는 것처럼 해주기 위해서 리액트용 주소를 처리해주는 방식입니다.

App.js 페이지를 다음과 같이 작업합니다.

import React from "react";
import { BrowserRouter, Route, Routes } from "react-router-dom";

import HomeView from "./views/HomeView";

const App = () => {
    return (
        <BrowserRouter>
            <Routes>
                <Route path="/" element={<HomeView />} />
            </Routes>
        </BrowserRouter>
    );
};

export default App;
  • import React from "react"; : React를 불러오는 구문으로, React 컴포넌트를 사용하기 위해 필요합니다.
  • import { BrowserRouter, Route, Routes } from "react-router-dom"; : react-router-dom에서 BrowserRouter, Route, Routes 컴포넌트를 불러옵니다. 이들은 React 애플리케이션에서 라우팅을 설정하는 데 사용됩니다.
  • import HomeView from "./views/HomeView"; : HomeView 컴포넌트를 불러옵니다. 이 컴포넌트는 루트 경로("/")에 해당하는 페이지를 표시하는 데 사용됩니다.
  • const App = () => { ... } : App 함수형 컴포넌트를 정의합니다. 이 컴포넌트는 React 애플리케이션의 진입점 역할을 합니다.
  • <BrowserRouter> : 라우팅을 설정하기 위한 최상위 컴포넌트인 BrowserRouter를 사용합니다. 이 컴포넌트는 웹 브라우저의 주소 표시줄과 상태를 동기화하여 현재 URL과 일치하는 컴포넌트를 렌더링합니다.
  • <Routes> : 라우트들을 정의하는 컨테이너인 Routes를 사용합니다. 이 컴포넌트는 다양한 </Route> 컴포넌트들을 포함하고 있습니다.
  • <Route path="/" element={<HomeView />} /> : / 경로에 해당하는 페이지를 HomeView 컴포넌트로 매핑합니다. 즉, 루트 경로(/)에 접속하면 HomeView 컴포넌트가 렌더링되도록 설정합니다.
  • export default App; : App 컴포넌트를 외부에서 사용할 수 있도록 내보내기(exports)합니다.

2_2. 컴퍼넌트 설정하기

리액트(React) 컴포넌트는 사용자 인터페이스를 구축하기 위한 재사용 가능한 블록입니다. 컴포넌트 기반 접근 방식을 통해 UI를 작성하며, 각 컴포넌트는 독립적으로 동작하고 상태(state)와 프로퍼티(props)를 가집니다. 이러한 컴포넌트들을 조합하여 인터페이스를 구현할 수 있습니다. 컴포넌트를 사용하는 이유는 다음과 같습니다.

  • 리액트 컴포넌트는 작고 독립적이며, 다른 컴포넌트와 조합하여 재사용이 가능합니다. 이로 인해 유지보수가 쉽고 효율적이며, 코드의 재사용성과 가독성이 높아집니다.
  • 리액트 애플리케이션은 여러 개의 컴포넌트들로 이루어져 있으며, 각 컴포넌트는 기능에 따라 분리됩니다. 이렇게 분리된 컴포넌트들은 상위 컴포넌트로부터 상태나 데이터를 전달받고, 하위 컴포넌트로 데이터를 전달합니다.
  • 리액트는 가상 DOM을 사용하여 성능을 최적화합니다. 가상 DOM은 실제 DOM과 동기화되어 있으며, 상태 변화에 따른 최소한의 변경만을 적용하여 DOM 조작을 최소화합니다.
  • 리액트 컴포넌트를 사용하여 웹 애플리케이션을 개발하면, UI의 재사용성과 유지보수성이 증가하며, 코드의 구조와 가독성이 향상됩니다

skip.jsx 메뉴는 다음과 같이 작성합니다.

import React from "react";

const Skip = () => {
    return (
        <div id="skip">

            <a href="#intro">소개 영역</a>
            <a href="#skill">스킬 영역</a>

            <a href="#footer">푸터 영역</a>

    );
};

export default Skip;

main.jsx에는 다음과 같이 작업해줍니다. 단순히 main 태그로 감싸주기 위함입니다. Main 컴포넌트를 사용하면 해당 컴포넌트로 감싼 내용을 main 태그로 래핑하여 페이지의 주요 콘텐츠를 표시할 수 있습니다. 이와 같이 컴포넌트를 정의하고 내보내는 방식으로, 리액트에서 재사용 가능한 컴포넌트를 만들 수 있습니다. 

import React from "react";

const Main = ({ children }) => {
    return (
        <main id="main" role="main">
            {children}
        </main>
    );
};

export default Main;

2_3. SCSS 설정하기

SCSS(Scalable and Cascading Style Sheets)는 CSS의 확장 문법으로, 더 유연하고 가독성이 높은 CSS 코드를 작성할 수 있도록 해주는 스타일 시트 언어입니다. SCSS는 Sass라는 CSS 전처리기의 하위 집합으로서, Sass의 기능과 문법을 확장한 것입니다. SCSS는 .scss 확장자를 가지며, 일반적인 CSS와 거의 동일한 문법을 사용할 수 있습니다. SCSS의 주요 특징과 기능은 다음과 같습니다.

  • 변수(Variables) : SCSS는 $ 기호를 사용하여 변수를 선언하고 사용할 수 있습니다. 이를 통해 색상, 폰트, 여백 등의 값들을 변수로 정의하여 재사용할 수 있습니다.
  • 중첩(Nesting) : SCSS는 CSS 규칙을 중첩해서 작성할 수 있습니다. 이를 통해 HTML의 계층 구조에 맞게 CSS를 구조화하고 가독성을 높일 수 있습니다.
  • 믹스인(Mixins) : SCSS는 @mixin 디렉티브를 사용하여 재사용 가능한 스타일 선언을 만들 수 있습니다. 이는 코드의 반복을 줄이고 유지보수를 편리하게 만듭니다.
  • 상속(Extend) : SCSS는 @extend 디렉티브를 사용하여 하나의 스타일 선언을 다른 스타일에 상속할 수 있습니다. 이는 중복 코드를 최소화하고 스타일을 모듈화하는데 도움을 줍니다.
  • 연산(Operations) : SCSS는 숫자, 색상 등의 값을 연산할 수 있는 기능을 제공합니다. 이로 인해 코드 작성이 더욱 편리해집니다.
  • 조건문(Conditional statements)과 반복문(Loops) : SCSS는 @if, @else, @for, @each 등의 제어문을 사용하여 동적으로 스타일을 생성할 수 있습니다.

vars.scss를 셋팅하겠습니다. 기존과 달라진 부분의 거의 없습니다. 주석표시와 미디어 쿼리를 root 안에 작업하였습니다. 디테일한 사용법은 섹션 작업할 때 좀 더 알아보겠습니다. 

:root {
    --mainEng-font: "Montserrat";
    --mainKor-font: "nanumSquareNeo";
    --mainNum-font: "gmarket";

    --subBg100: #cdc0b1;
    --subBg200: #afa395;
    --subBg300: #81887c;
    --subBg400: #afa7a2;
    --subBg500: #a6afa2;

    --white: #fff;
    --black: #000;
    --black100: #2b2b2b;
    --black200: #434343;
    --black300: #686868;
    --black400: #e0e0e0;

    // 기본 폰트 설정
    font-family: var(--mainEng-font), var(--mainKor-font);
    font-size: 16px;
    line-height: 1.5;
    font-weight: 400;

    // 폰트를 부드럽게 렌더링하기 위한 속성

    -webkit-text-size-adjust: 100%;

    @media (max-width: 800px) {
        :root {
            font-size: 14px;
            line-height: 1.4;
        }
    }
}

body {
    background-color: var(--mainBg-color);

3. 마무리

잘 정리하셨나요? 컴포넌트는 웹 애플리케이션을 구성하는 작은 조각들로, 각 컴포넌트는 독립적으로 작동하며 필요에 따라 재사용할 수 있습니다. 컴포넌트를 사용하면 UI를 구성하는 데 있어서 논리적이고 구조적으로 접근할 수 있으며, 코드를 더욱 간결하고 가독성 좋게 작성할 수 있습니다.

SCSS는 CSS의 확장 문법으로, CSS의 기능을 확장하여 더욱 유연하고 편리한 스타일 시트를 작성할 수 있습니다. SCSS를 사용하면 변수, 중첩, 믹스인, 상속 등의 기능을 사용하여 스타일을 구성할 수 있으며, 코드의 반복을 줄이고 유지보수를 용이하게 만들 수 있습니다. 또한, SCSS를 사용하면 CSS에 없는 조건문과 반복문을 사용하여 동적인 스타일을 생성할 수 있습니다.

처음에는 새로운 개념들이 추가되어 혼란스러울 수 있지만, 익숙해지면 더 쉽게 사용할 수 있습니다. 코드 작성에 익숙해지기 위해서는 계속해서 연습하고 참고 자료를 활용하면 좋습니다. 기존 개념과 새로운 개념을 조합하여 더 효율적이고 유연한 웹 애플리케이션을 구축할 수 있습니다. 오늘도 수고하셨습니다! 계속해서 열심히 학습하고 개발하시면 더 나은 개발자로 성장하실 수 있을 것입니다. 궁금한 점이나 도움이 필요하면 언제든지 댓글주세요! 🤭
